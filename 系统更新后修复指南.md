# 🔄 系统更新后修复指南

## 📍 问题说明

**Ubuntu 系统更新后出现 502 错误**，通常是因为：

1. ✅ **系统重启** → PM2 应用没有自动启动
2. ✅ **Node.js 版本变化** → 应用可能需要重新构建
3. ✅ **依赖库更新** → 原生模块（如 better-sqlite3）需要重新编译
4. ✅ **服务配置丢失** → PM2 开机自启配置可能失效

---

## 🚀 快速修复步骤

### 步骤 1：检查 PM2 状态

```bash
ssh songtuo@192.168.13.44 "pm2 list"
```

**如果看到应用状态是 `stopped` 或没有应用：**

### 步骤 2：重启应用

```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
cd /home/songtuo/下载/complete-project/.next/standalone
pm2 delete personality-assessment 2>/dev/null || true
pm2 start node --name "personality-assessment" -- server.js
pm2 save
ENDSSH
```

### 步骤 3：重新配置 PM2 开机自启

```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
# 生成开机自启脚本
pm2 startup

# 保存当前 PM2 进程列表
pm2 save
ENDSSH
```

**注意**：`pm2 startup` 会输出一个命令，需要以 root 权限执行。复制输出的命令，然后执行：

```bash
ssh songtuo@192.168.13.44
# 然后执行 pm2 startup 输出的命令（通常是 sudo 开头的）
```

## 🔧 常见问题和解决方案

### 问题 1：PM2 应用未运行

**原因**：系统重启后，PM2 没有自动启动应用

**解决**：
```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
cd /home/songtuo/下载/complete-project/.next/standalone
pm2 start node --name "personality-assessment" -- server.js
pm2 save
pm2 startup  # 重新配置开机自启
ENDSSH
```

---

### 问题 2：Node.js 版本变化

**原因**：系统更新可能改变了 Node.js 版本

**检查**：
```bash
ssh songtuo@192.168.13.44 "node --version"
```

**如果版本变化**：
```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
# 检查当前版本
node --version

# 如果需要，重新安装依赖和构建
cd /home/songtuo/下载/complete-project
npm install
npm run build
ENDSSH
```

---

### 问题 3：原生模块需要重新编译

**原因**：系统更新后，原生模块（如 better-sqlite3）需要重新编译

**解决**：
```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
cd /home/songtuo/下载/complete-project/.next/standalone

# 重新安装原生模块
npm install better-sqlite3 --production --no-save

# 重启应用
pm2 restart personality-assessment
ENDSSH
```

---

### 问题 4：PM2 开机自启配置失效

**原因**：系统更新可能重置了 systemd 服务配置

**解决**：
```bash
ssh songtuo@192.168.13.44 << 'ENDSSH'
# 重新生成开机自启配置
pm2 startup

# 保存当前进程列表
pm2 save
ENDSSH
```

**然后执行 `pm2 startup` 输出的命令**（需要 root 权限）。

## 📋 完整修复脚本

创建一个一键修复脚本：

```bash
#!/bin/bash

echo "🔄 系统更新后修复脚本"
echo ""

SERVER_USER="songtuo"
SERVER_HOST="192.168.13.44"
APP_DIR="/home/songtuo/下载/complete-project"

echo "1️⃣  检查系统状态..."
ssh ${SERVER_USER}@${SERVER_HOST} << 'ENDSSH'
echo "Node.js 版本: $(node --version)"
echo "PM2 状态:"
pm2 list
ENDSSH

echo ""
echo "2️⃣  检查应用状态..."
HTTP_CODE=$(ssh ${SERVER_USER}@${SERVER_HOST} "curl -s -o /dev/null -w '%{http_code}' http://localhost:3000 2>/dev/null || echo '000'")

if [ "$HTTP_CODE" = "200" ]; then
    echo "✅ 应用正常运行"
else
    echo "❌ 应用无响应，开始修复..."

    echo ""
    echo "3️⃣  重启应用..."
    ssh ${SERVER_USER}@${SERVER_HOST} << 'ENDSSH'
cd /home/songtuo/下载/complete-project/.next/standalone
pm2 delete personality-assessment 2>/dev/null || true
pm2 start node --name "personality-assessment" -- server.js
pm2 save
ENDSSH

    echo ""
    echo "4️⃣  重新安装原生模块（如果需要）..."
    ssh ${SERVER_USER}@${SERVER_HOST} << 'ENDSSH'
cd /home/songtuo/下载/complete-project/.next/standalone
npm install better-sqlite3 --production --no-save 2>&1 | tail -5
ENDSSH

    echo ""
    echo "5️⃣  等待应用启动..."
    sleep 10

    echo ""
    echo "6️⃣  验证修复..."
    HTTP_CODE=$(curl -s -o /dev/null -w '%{http_code}' http://localhost:3000 2>/dev/null || echo '000')
    if [ "$HTTP_CODE" = "200" ]; then
        echo "✅ 修复成功！"
    else
        echo "❌ 修复失败，请查看日志："
        ssh ${SERVER_USER}@${SERVER_HOST} "pm2 logs personality-assessment --lines 30 --nostream"
    fi
fi

echo ""
echo "7️⃣  配置 PM2 开机自启..."
echo "执行以下命令配置开机自启："
ssh ${SERVER_USER}@${SERVER_HOST} "pm2 startup"
echo ""
echo "⚠️  请复制上面输出的命令，然后以 root 权限执行"
```
