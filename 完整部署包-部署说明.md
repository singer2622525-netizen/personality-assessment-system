## 🚀 部署步骤

### 步骤1：上传部署包到服务器

```bash

```

```bash
ssh songtuo@192.168.13.44
```

### 步骤3：备份现有项目（重要！）

```bash

cd ~/personality-assessment  # 或您的项目路径

# 备份整个项目
```

### 步骤4：解压并部署

```bash

tar -xzf 完整部署包.tar.gz

# 进入解压后的目录
cd 招聘人格评测结果智能分析

# 方法1：替换整个项目（推荐，如果项目结构相同）
# 停止当前服务
pm2 stop all  # 或 Ctrl+C 停止 npm start

# 备份现有项目后，复制新文件
# 注意：保留 data/ 目录（数据库文件）
cp -r app lib *.json *.js *.ts ~/personality-assessment/

# 方法2：手动复制关键文件
# 如果只想更新修复的文件，可以只复制：
# - app/api/sessions/ 目录（修复了params问题）
# - app/admin/dashboard/ 目录（修复了字段名问题）
# - app/layout.tsx, app/page.tsx（公司名称更新）
```

### 步骤5：安装依赖（如果需要）

```bash
cd ~/personality-assessment
npm install
```

### 步骤6：重新构建项目

```bash
npm run build
```

### 步骤7：重启服务

**如果使用PM2：**
```bash
pm2 restart all
# 或
pm2 restart personality-assessment
```

**如果直接运行：**
```bash
npm start
```

## ⚠️ 注意事项

### 1. 数据库文件
- **重要**：部署时不要覆盖 `data/assessment.db` 文件
- 这个文件包含所有评测数据
- 如果覆盖，会丢失所有历史数据

### 2. 环境变量
- 如果设置了环境变量（如 `JWT_SECRET`），确保在部署后仍然有效

### 3. 服务管理
- 如果使用PM2，确保服务名称正确
- 如果不确定，使用 `pm2 list` 查看

### 4. 端口占用
- 确保端口3000没有被其他程序占用
- 可以使用 `netstat -tulpn | grep 3000` 检查
