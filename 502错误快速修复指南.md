# 🔧 502 Bad Gateway 错误快速修复指南

## 📍 问题说明

**502 Bad Gateway** 错误表示：
- ✅ Cloudflare 正常（绿色）
- ✅ 浏览器正常（绿色）
- ❌ **服务器端应用无法响应**（红色）

这通常是因为：
1. Next.js 应用（PM2）没有运行
2. 应用崩溃或启动失败
3. 端口 3000 没有监听

## 🔍 常见问题排查

### Q1: PM2 应用一直启动失败？

**检查项目目录：**
```bash
cd /home/songtuo/下载/complete-project
ls -la
```

**检查是否有构建产物：**
```bash
ls -la .next/standalone
```

**如果没有 standalone 目录，需要重新构建：**
```bash
cd /home/songtuo/下载/complete-project
npm install
npm run build
```

**然后重新启动：**
```bash
cd .next/standalone
pm2 delete personality-assessment
pm2 start node --name "personality-assessment" -- server.js
pm2 save
```

---

### Q2: 端口 3000 被占用？

**检查端口占用：**
```bash
netstat -tulpn | grep 3000
# 或
ss -tlnp | grep 3000
```

**如果被占用，找到进程并杀死：**
```bash
# 找到占用端口的进程 ID
lsof -i :3000

# 杀死进程（替换 PID 为实际进程 ID）
kill -9 <PID>
```

**然后重新启动 PM2：**
```bash
pm2 restart personality-assessment
```

---

### Q3: 应用启动但无法响应？

**检查应用日志：**
```bash
pm2 logs personality-assessment --lines 100
```

**常见错误：**
- `better-sqlite3` 模块错误 → 需要重新安装原生模块
- 数据库文件权限问题 → 检查 `data/` 目录权限
- 端口冲突 → 检查端口占用

---

### Q4: 修复后仍然 502？

**可能原因：**
1. **Cloudflare Tunnel 未运行** → 检查 Tunnel 服务
2. **浏览器缓存** → 清除缓存或使用无痕模式
3. **DNS 缓存** → 等待几分钟或刷新 DNS

**检查 Cloudflare Tunnel：**
```bash
# 在服务器上执行
ps aux | grep cloudflared

# 如果没有运行，启动 Tunnel
sudo systemctl restart cloudflared
# 或
cloudflared tunnel --url http://localhost:3000 run personality-assessment
```
