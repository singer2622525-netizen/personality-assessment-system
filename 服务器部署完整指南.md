## 🌐 域名配置

### 步骤 1: 购买域名
推荐域名注册商：
- 阿里云
- 腾讯云
- 西部数码
- Godaddy

### 步骤 2: 解析域名到服务器 IP

#### 阿里云 DNS 配置示例：

1. **登录阿里云控制台** → 域名 → 解析

2. **添加解析记录**：
   ```
   记录类型：A
   主机记录：@ 或 www
   解析线路：默认
   记录值：你的服务器公网IP（例如：123.45.67.89）
   TTL：10分钟
   ```

3. **如果使用子域名**（如 assessment.example.com）：
   ```
   记录类型：A
   主机记录：assessment
   解析线路：默认
   记录值：你的服务器公网IP
   TTL：10分钟
   ```

### 步骤 3: 验证解析
```bash
# 在本地电脑运行
ping yourdomain.com

# 或使用 nslookup
nslookup yourdomain.com
```

✅ 能解析到服务器IP即配置成功

## 🚀 Docker 部署

### 步骤 1: 上传项目文件

#### 方法 A: 使用 Git（推荐）
```bash
# 在服务器上
cd /opt/personality-assessment
git clone https://github.com/your-username/personality-assessment.git .

# 如果没有Git，先安装
yum install -y git  # CentOS
apt install -y git   # Ubuntu
```

#### 方法 B: 上传压缩包
1. 在本地电脑压缩项目目录为 `project.tar.gz`
2. 上传到服务器：
```bash
# 使用 scp（在本地电脑运行）
scp project.tar.gz root@your-server-ip:/opt/personality-assessment/

# 在服务器解压
cd /opt/personality-assessment
tar -xzf project.tar.gz
```

### 步骤 2: 配置环境变量

创建 `.env.production` 文件：
```bash
cd /opt/personality-assessment
nano .env.production
```

添加以下内容：
```env
NODE_ENV=production
PORT=3000
```

保存并退出（Ctrl+X, Y, Enter）

### 步骤 3: 构建并启动容器
```bash
cd /opt/personality-assessment

# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 查看运行状态
docker-compose ps
```

✅ 服务启动后，在浏览器访问 `http://your-server-ip:3000` 应该能看到系统

## 🔧 Nginx 配置

### 步骤 1: 配置 Nginx 反向代理

编辑 Nginx 配置：
```bash
nano /etc/nginx/sites-available/personality-assessment
```

**Ubuntu/Debian** 用户使用 `sites-available`，**CentOS** 用户直接编辑：
```bash
nano /etc/nginx/conf.d/personality-assessment.conf
```

添加以下内容（**修改成你的域名**）：
```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;  # 改成你的域名
    
    # 反向代理到 Docker 容器
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

### 步骤 2: 启用站点（仅 Ubuntu/Debian）
```bash
ln -s /etc/nginx/sites-available/personality-assessment /etc/nginx/sites-enabled/
```

### 步骤 3: 测试配置并重启
```bash
# 测试配置文件
nginx -t

# 重启 Nginx
systemctl restart nginx

# 设置开机自启
systemctl enable nginx
```

✅ 现在可以通过域名访问：`http://yourdomain.com`

## 🔒 SSL 证书配置（HTTPS）

### 方法 1: 使用 Let's Encrypt（免费，推荐）

#### 安装 Certbot
```bash
# Ubuntu/Debian
apt install -y certbot python3-certbot-nginx

# CentOS
yum install -y epel-release
yum install -y certbot python3-certbot-nginx
```

#### 获取证书
```bash
# 自动配置证书和 Nginx
certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 按提示操作，输入邮箱地址
```

证书会自动配置，现在可以通过 HTTPS 访问：`https://yourdomain.com`

#### 设置自动续期
证书有效期 90 天，Certbot 会自动续期：
```bash
# 测试续期
certbot renew --dry-run

# 证书会自动续期，无需手动操作
```

### 方法 2: 使用其他 SSL 证书

如果你有其他证书（如阿里云、腾讯云购买的证书）：

1. **上传证书文件**到服务器：
```bash
mkdir -p /etc/nginx/ssl
# 上传 cert.pem 和 key.pem 到这个目录
```

2. **修改 Nginx 配置**：
```bash
nano /etc/nginx/sites-available/personality-assessment
```

3. **添加 SSL 配置**（参考以下章节）

### 完整的 HTTPS 配置示例

```nginx
# 重定向 HTTP 到 HTTPS
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;
    
    # SSL 证书路径（Let's Encrypt 自动配置）
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    
    # 使用其他证书时替换为：
    # ssl_certificate /etc/nginx/ssl/cert.pem;
    # ssl_certificate_key /etc/nginx/ssl/key.pem;
    
    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    
    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer-when-downgrade" always;
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;
    
    # 反向代理
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

保存后重新加载 Nginx：
```bash
nginx -t
systemctl reload nginx
```

## 🔥 防火墙配置

确保服务器开放必要端口：

```bash
# CentOS/FirewallD
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload

# Ubuntu/UFW
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 22/tcp  # SSH
ufw enable

# 云服务器还需要在云控制台配置安全组
# 开放端口：80, 443, 22
```

## ❓ 常见问题

### Q1: 访问域名显示 "无法访问此网站"
**解决方案**：
1. 检查域名 DNS 解析是否正确
2. 检查服务器防火墙和安全组是否开放 80/443 端口
3. 在服务器上测试：`curl http://localhost:3000`

### Q2: 访问显示 502 Bad Gateway
**解决方案**：
```bash
# 检查 Docker 容器是否运行
docker-compose ps

# 重启服务
docker-compose restart

# 查看日志排查错误
docker-compose logs
```

### Q3: HTTPS 证书过期
**解决方案**：
```bash
# Let's Encrypt 证书续期
certbot renew

# 重新加载 Nginx
systemctl reload nginx
```

### Q4: 忘记管理员密码
**解决方案**：
系统使用 localStorage 存储数据，清除浏览器数据后重新注册

### Q5: 如何备份数据
**解决方案**：
```bash
# 备份数据目录
tar -czf backup-$(date +%Y%m%d).tar.gz /opt/personality-assessment/data

# 恢复数据
tar -xzf backup-YYYYMMDD.tar.gz
```

### Q6: 如何更新系统
```bash
cd /opt/personality-assessment
git pull  # 拉取最新代码
docker-compose build --no-cache  # 重新构建
docker-compose up -d  # 重启服务
```
