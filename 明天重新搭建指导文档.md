# 5型人格评测系统 - 明天重新搭建指导文档

## 项目概述
基于 Next.js 14 的5型人格评测系统，专为招聘场景设计，包含应聘者评测和管理员后台功能。

## 核心功能清单

### 1. 应聘者评测流程
- **开始页面** (`/assessment/start`)
  - 姓名、邮箱、手机、岗位信息填写
  - **重要**：岗位字段必须是文本输入框（input type="text"），不是下拉选择
  - 表单验证：中文姓名2-10字、手机号11位、邮箱格式
  - 生成唯一会话ID：`generateUniqueId(name, phone)`

- **评测页面** (`/assessment/[sessionId]`)
  - 50道5型人格题目
  - 实时进度保存
  - 候选人信息显示确认
  - 自动跳转下一题

- **结果页面** (`/assessment/[sessionId]/results`)
  - 雷达图可视化
  - 详细分析报告
  - PDF下载功能
  - 管理员/应聘者不同视图

### 2. 管理员后台功能
- **登录页面** (`/admin/login`)
  - 多角色管理员系统
  - 预设账户（不显示在页面上）

- **仪表板** (`/admin/dashboard`)
  - 评测记录列表
  - 搜索和筛选
  - 统计信息展示
  - 创建评测任务

- **对比分析** (`/admin/compare`)
  - 多候选人对比
  - 雷达图对比
  - 详细分数对比

- **统计分析** (`/admin/statistics`)
  - 整体统计趋势
  - 岗位分布分析
  - 月度统计图表

- **创建评测** (`/admin/create`)
  - 候选人信息录入
  - 评测链接生成

## 今天解决的关键问题

### 1. 数据存储问题 ⚠️
**问题**：会话数据存储键名不一致导致"会话不存在"
**解决方案**：
- 统一使用 `assessmentSession_${sessionId}` 键名
- 开始页面：`localStorage.setItem(\`assessmentSession_${sessionId}\`, data)`
- 评测页面：`localStorage.getItem(\`assessmentSession_${sessionId}\`)`
- 结果页面：`localStorage.getItem(\`assessmentSession_${sessionId}\`)`

### 2. 导入错误 ⚠️
**问题**：`import { QUESTIONS } from '@/lib/questions'` 文件不存在
**解决方案**：
- 将 `QUESTIONS` 数组移到 `lib/types.ts` 中
- 统一导入：`import { QUESTIONS } from '@/lib/types'`

### 3. 岗位字段类型 ⚠️
**问题**：岗位字段使用下拉选择，与Mac版本不一致
**解决方案**：
- 使用 `<input type="text">` 而不是 `<select>`
- 允许用户自由输入岗位名称

### 4. ID生成问题 ⚠️
**问题**：`generateUniqueId` 可能生成包含特殊字符的ID
**解决方案**：
```typescript
export function generateUniqueId(name: string, phone: string): string {
  const timestamp = Date.now().toString(36)
  const nameHash = name.slice(0, 2)
  const phoneHash = phone.slice(-4)
  return `${timestamp}_${nameHash}_${phoneHash}`
}
```

### 5. 依赖缺失 ⚠️
**问题**：构建失败 `Module not found: Can't resolve 'tailwind-merge'`
**解决方案**：
- 在 `package.json` 中添加：
```json
{
  "dependencies": {
    "clsx": "^2.0.0",
    "tailwind-merge": "^2.0.0"
  }
}
```

### 6. 类型定义问题 ⚠️
**问题**：`Answer` 和 `Question` 接口属性不匹配
**解决方案**：
```typescript
export interface Question {
  id: number
  text: string
  trait: PersonalityDimension
  reverse: boolean
}

export interface Answer {
  questionId: number
  answer: number
}
```
