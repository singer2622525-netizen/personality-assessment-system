# 📋 问题说明 - 域名解析 vs 代码问题

## ❌ 误解澄清

**域名解析和提交评测卡住的问题没有关系！**

这是一个**代码层面的问题**，与域名、网络、公网访问都无关。

---

## 🔍 问题本质

### 问题类型：代码错误
- **位置**：API 路由处理函数
- **原因**：Next.js 14 的 API 路由参数处理方式变更
- **影响**：无论在哪里运行（本地、服务器、公网），都会出现

### 与域名解析无关
- ✅ **本地运行**（localhost:3000）- 会出现这个问题
- ✅ **服务器本地**（192.168.13.44:3000）- 会出现这个问题
- ✅ **公网访问**（域名解析后）- 也会出现这个问题

**结论**：无论是否解析域名，这个问题都存在！

---

## 🎯 为什么会出现这个问题？

### Next.js 14 的变更

在 Next.js 14 的 App Router 中：
- **旧版本**（Next.js 13）：`params` 是普通对象
  ```typescript
  { params }: { params: { sessionId: string } }
  // 可以直接使用 params.sessionId
  ```

- **新版本**（Next.js 14）：`params` 是 Promise
  ```typescript
  { params }: { params: Promise<{ sessionId: string }> }
  // 必须先 await params 才能使用
  ```

### 问题表现

当代码直接使用 `params.sessionId` 时：
- `params` 是一个 Promise 对象，不是普通对象
- `params.sessionId` 是 `undefined`
- API 无法获取正确的 `sessionId`
- 数据库更新失败
- 前端等待响应，最终超时或卡住
