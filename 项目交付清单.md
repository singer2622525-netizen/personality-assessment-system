## 📦 如何打包项目给服务器

### 方法 1: Git 克隆（推荐）
```bash
# 在服务器上
cd /opt
git clone <repository-url> personality-assessment

```

### 方法 2: 上传压缩包
```bash
# 在本地电脑压缩（排除不必要文件）
tar -czf personality-assessment.tar.gz \
  --exclude='node_modules' \
  --exclude='.next' \
  --exclude='.git' \
  --exclude='*.tar.gz' \
  --exclude='complete-project' \
  .

# 上传到服务器
scp personality-assessment.tar.gz root@server:/tmp/

# 在服务器解压
ssh root@server
mkdir -p /opt/personality-assessment
cd /opt/personality-assessment
tar -xzf /tmp/personality-assessment.tar.gz
```

将项目推送到远程仓库，然后在服务器克隆

## 🌐 域名和访问配置

### 1. 域名解析

在域名管理平台（如阿里云、腾讯云）添加 A 记录：

```
类型：A
主机记录：@（或 www，或其他子域名）
记录值：服务器公网IP
TTL：600
```

### 2. 防火墙配置

**云服务器**：在云平台控制台配置安全组，开放端口：
- 22 (SSH)
- 80 (HTTP)
- 443 (HTTPS)

**本地服务器**：配置防火墙规则
```bash
# CentOS/FirewallD
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload

# Ubuntu/UFW
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 22/tcp
ufw enable
```

## 🆘 故障排查

### 问题 1: 无法访问网站
**解决步骤**:
```bash
# 1. 检查服务是否运行
docker-compose ps

# 2. 检查容器日志
docker-compose logs

# 3. 检查 Nginx 配置
nginx -t

# 4. 检查域名解析
nslookup yourdomain.com
```

### 问题 2: 502 Bad Gateway
**解决步骤**:
```bash
# 1. 重启服务
docker-compose restart

# 2. 检查端口是否被占用
netstat -tulpn | grep 3000

# 3. 检查防火墙
iptables -L -n
```

### 问题 3: SSL 证书失效
**解决步骤**:
```bash
# 续期证书
certbot renew

# 重启 Nginx
systemctl reload nginx
```
